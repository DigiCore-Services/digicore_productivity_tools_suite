# Ticket: KMS note favorites (IPC + UI)

**Type:** Feature (P0)  
**Related:** `digicore/docs/kms-ux-feature-gaps-and-opportunities-2026-04.md` (gap inventory: section 3 P0 item 1, Favorites).  
**Estimate:** Small (1 backend method + repository update + 2–3 UI surfaces + bindings sync)

---

## Problem

`kms_notes.is_favorite` and `KmsNoteDto.is_favorite` exist, but there is no RPC to change the flag and the **Favorites** sidebar view is a placeholder. Users cannot star notes or see a favorites list.

---

## Scope (in)

- One IPC procedure: set favorite boolean for a note identified by the same path convention as `kms_save_note` / `kms_list_notes`.
- Repository function to persist `is_favorite` in SQLite for that row (by path).
- **Favorites** view in KMS: list favorite notes, open on click, empty state when none.
- **Editor** toolbar control: toggle favorite for the active note (reflects current state, optimistic or refresh-after-success).

## Scope (out / follow-ups)

- ~~Star from **FileExplorer** context menu~~ **Done** (`FileExplorer.tsx` + `onSetNoteFavorite` / `notes` from `KmsApp.tsx`).
- ~~Favorites sort order beyond stable default~~ **Done in UI:** manual order via drag handle in Favorites list; persisted in `localStorage` (`kms-favorite-path-order-v1`).
- Syncing favorites across devices (vault is local; no cloud spec).

## API shape (recommended)

```rust
// api.rs (trait excerpt)
async fn kms_set_note_favorite(path: String, favorite: bool) -> Result<(), String>;
```

- **path:** Same semantics as `kms_load_note` / `kms_save_note` (whatever the UI already passes today for `activeNote.path`).
- **favorite:** `true` = star, `false` = unstar.

---

## Acceptance criteria

1. **IPC persistence:** After `kms_set_note_favorite(path, true)`, a subsequent `kms_list_notes()` returns a `KmsNoteDto` for that path with `is_favorite === true`; `false` clears it.
2. **Unknown path:** Calling the RPC for a path not present in `kms_notes` returns an error (structured KMS IPC error with non-empty message); UI shows a toast and does not claim success.
3. **Favorites view:** With at least one favorite, selecting **Favorites** in the KMS sidebar shows those notes (title/path consistent with search/explorer). Clicking an item opens that note in the editor (same behavior as choosing from explorer).
4. **Favorites empty state:** With zero favorites, the Favorites view shows a short, non-placeholder message (e.g. “No favorites yet” + hint to use the star in the editor).
5. **Editor toggle:** While a note is open, the star control reflects `is_favorite`; clicking it flips state, calls the new RPC, and on success the control and Favorites list stay in sync (via `refreshNotes` or equivalent).
6. **Explorer menu:** For a note row, the overflow menu offers **Add to favorites** / **Remove from favorites**; after success, the menu label/icon matches `kms_list_notes` (same `notes` state as Favorites view).
7. **Build:** `cargo check` (src-tauri) passes; `npm run build` (tauri-app) passes after `bindings.ts` is updated to match codegen for `ARGS_MAP` / `Router`.

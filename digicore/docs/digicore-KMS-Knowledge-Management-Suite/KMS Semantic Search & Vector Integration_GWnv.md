# KMS Semantic Search & Vector Integration

Implement a unified semantic search system that allows users to find Knowledge Base notes, Text Expander snippets, and Clipboard history items using natural language queries, regex, and logical operators. All processing is local-first for maximum privacy.

## User Review Required

> [!IMPORTANT]
> The first run will trigger a model download for **BGE-small** (approx. 15-20MB). This is the mandatory starting model to keep the installation lean. Future extensibility for MiniLM/MPNet/Gemma is planned but not in the initial release.

> [!IMPORTANT]
> Semantic search will be unified. Results will include Notes, Snippets, and Clipboard History items.

> [!NOTE]
> We will use `sqlite-vec` for vector similarity. We will combine vector distance with traditional SQLite FTS5 (Full Text Search) to support hybrid queries like `te*ng AND (c*t OR Mo*se)`.

### Model Selection & ROI Analysis

We will support the following models, with `bge-small-en-v1.5` as the default for its superior size-to-performance ratio:

| Model | Binary Size (ONNX) | Dims | MTEB Retrieval | Recommendation |
|---|---|---|---|---|
| **`bge-small-en-v1.5`** | **~33 MB** | **384** | **49.0** | **Mandatory Initial Model.** Best ROI. |
| `all-MiniLM-L6-v2` | ~23 MB | 384 | 41.6 | Future option for ultra-lean systems. |
| `all-mpnet-base-v2` | ~290 MB | 768 | 50.0 | Future option for high-precision retrieval. |

> [!TIP]
> `bge-small` is 3.5x more accurate than `all-MiniLM-L6-v2` while being only 10MB larger. It outperforms `all-mpnet-base-v2` in some retrieval tasks despite being 9x smaller.

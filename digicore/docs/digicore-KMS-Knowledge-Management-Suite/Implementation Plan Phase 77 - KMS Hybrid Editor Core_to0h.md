# Implementation Plan: Phase 77 - KMS Hybrid Editor Core

This phase focuses on building the core editing experience for the DigiCore Knowledge Management Suite (KMS). We will implement a dual-mode editor that allows users to switch between a rich-text WYSIWYG view (TipTap) and a raw Markdown source view (CodeMirror).

## Proposed Changes

### [Frontend] Editor Components & State
#### [NEW] [KmsEditor.tsx](file:///c:/Users/pinea/Scripts/AHK_AutoHotKey/digicore/tauri-app/src/components/kms/KmsEditor.tsx)
- Create a reusable `KmsEditor` component.
- Implement a `mode` state (`wysiwyg` | `source`).
- Integrate **TipTap** for the `wysiwyg` mode.
- Integrate **CodeMirror** for the `source` mode.
- Use `tiptap-markdown` to ensure seamless conversion between rich text and Markdown.

#### [MODIFY] [KmsApp.tsx](file:///c:/Users/pinea/Scripts/AHK_AutoHotKey/digicore/tauri-app/src/KmsApp.tsx)
- Add state to track the `activeNote` (path, content).
- Render `KmsEditor` in the main content area when a note is selected.
- Add "Save" and "Mode Toggle" buttons to the editor header.

### [Backend] File & Index Management
#### [MODIFY] [api.rs](file:///c:/Users/pinea/Scripts/AHK_AutoHotKey/digicore/tauri-app/src-tauri/src/api.rs)
- Add `kms_load_note(path: String)`: Reads file from the vault.
- Add `kms_save_note(path: String, content: String)`: Writes file and updates the `kms_notes` SQLite table (title, preview, last_modified).
- Add `kms_list_notes()`: Returns a list of notes from the `kms_notes` table for the sidebar.

## Verification Plan

### Automated Tests
- No automated tests for UI components yet. I will focus on visual and functional manual verification.

### Manual Verification
1.  **Dependency Check**: Run `npm install` and verify TipTap is correctly added.
2.  **Mode Switching**: Open a note, switch from WYSIWYG to Source, change text, and switch back. Verify the content is preserved and correctly formatted.
3.  **Persistence**: Save a note in the KMS. Close the window, reopen, and verify the changes are loaded correctly.
4.  **Database Sync**: Verify that saving a note updates the `content_preview` and `last_modified` fields in the `kms_notes` table using a SQLite browser.
5.  **Markdown Fidelity**: Open the `.md` file in an external editor (like Notepad++) to ensure the Markdown generated by TipTap is clean and standard.
